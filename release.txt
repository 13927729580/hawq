Release-0_0_0:  26/Sep/2005  builder
    Dry Run for Oct 15 2005 Release

Fixes: mpp-55, mpp-63. Also added ctrl-c support and disabled COPY in 
    read only transactions.
Added new GUC variable Gp_max_packet_size.  This specifies the largest packet
	size that the interconnect will send messages with.  Like Mpp_session_role, 
	this can only be set before startup.  The value that is set on the QD is
	always set on all of the QEs as part of the QE connection process. The 
	default value is 8 kBytes and valid are from 512-64k bytes.  The GUC 
	Gp_max_tuple_chunk_size has been removed.  The tuple chunk size is now
	derived from Gp_max_packet_size.
COPY command integration with DTM completed. COPY now relies on the 
    transaction manager to issue a ROLLBACK if the dispatcher throws
    an error.
NAME datatype is now valid for hashing.
Major simplifications to the cdbloader front end: removed the hashing
    logic, removed all communications to the admin schema. The loader
    now connects only to the entry node, and the entry node information
    is still passed into the loader from the command line like before.
Added support for hashing on FLOAT4 and FLOAT8. Also added the option
    to select either the FNV1 or FNV1A hash algorithm, so that they
    should both be tested. Reduction methods are "lazy-mod" and "bit
    mask". Bit mask reduction is selected when the number of segments
    is a power of 2, and Lazymod otherwize. This is done for performance
    sake.
CDB Hash API added. So far support for hashing on INT2,INT4,INT8,
    TEXT,VARCHAR(n),CHAR(n) is added. Corresponding manual tests
    are located in libcdbtest. 
MPPEXEC command added.
    The command MPPEXEC is for internal use  only.
Log4j is the new CDB-Loader logger. Logging is now configurable from
    a properties config file and also some config options exist in the
    command line argument options (command line on higher priority).
Data load rate (MB/sec) added to the end of load statistics in the CDB-Loader.
CDB-Loader code changes that accomodate the change from a "cdb-head and
    tails" in CDB1.x to an all equal segment database cluster.
CDB2 admin schema version 1 support added to the CDB-Loader. Besides
    the support for this schema version, other schema versions support
    could be added in the future and loaded automatically. The loader
    now asks the admin schema which version it is and loads the appropriate
    set of queries for that particular version. 
Added traffic control and preliminary parallelization to SQL DML.
    CDB now analyzes and, in some cases, parallelizes the SQL DML commands,
    SELECT, INSERT, UPDATE, and DELETE. In addition, it recognizes and 
    rejects commands that should be parallelized but that the current code 
    can't handle.  Parallelized plans still execute in a single segment
    database, but the effects of parallelization may be observed via the
    EXPLAIN command.
Fixed some traffic control and message issues (CDB-5)
    ALTER messages now distinguish failures due to unsupported objects
    (like ALTER TABLE) versus unsupported operations  (like ALTER OWNER).
    Clarified some other messages and aligned the program behavior with
    the documented behavior vis a vis rejection versus interim rejection
Special handling of SQL commands for CDB users.
    Added "traffic control" for SQL commands to issue errors or warnings 
    as appropriate when Mpp_role=dispatch.  Modified PostgreSQL utilities 
    (e.g, initdb, createdb) to override external specification of 
    Mpp_session_role, thus preserving their old behavior.
New run-time variables Mpp_session_role and Mpp_role.
    The run-time parameters Mpp_session_role and Mpp_role report and provide 
    control over the role assumed by a postgres process.  The roles are 
    dispatch (process acts as a parallel SQL dispatcher), execute (process 
    acts as a parallel SQL executor), and utility (process acts as a simple 
    SQL engine. They are initialized to the same value at connection time and 
    are local to the resulting session. A super user can modify Mpp_role via 
    the SET command, but Mpp_session_role may not be modified.
